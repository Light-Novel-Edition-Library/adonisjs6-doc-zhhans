# 配置 (Configuration)

您的 AdonisJS 应用程序的配置文件存储在 `config` 目录中。一个全新的 AdonisJS 应用程序带有一些预先存在的文件，这些文件由框架核心和已安装的软件包使用。

您可以根据应用程序的需要在 `config` 目录中随意创建额外的文件。

:::note

我们建议使用[环境变量](https://www.google.com/search?q=./environment_variables.md)来存储秘密信息和特定于环境的配置。

:::

## 导入配置文件 (Importing config files)

您可以使用标准的 JavaScript `import` 语句在应用程序代码库中导入配置文件。例如：

```ts
import { appKey } from '#config/app'
```

```ts
import databaseConfig from '#config/database'
```

当您导入配置文件时，您可以访问导出的值。在大多数情况下，这些导出是 [`ConfigProvider`](https://www.google.com/search?q=../concepts/config_providers.md) 实例，因此**不建议**直接使用它们的值。相反，应从[解析后的配置](https://www.google.com/search?q=../concepts/config_providers.md%23how-do-i-access-the-resolved-config)中读取值。

## 使用配置服务 (Using the config service)

配置服务 (config service) 提供了另一种用于读取配置值的 API。在以下示例中，我们使用配置服务读取存储在 `config/app.ts` 文件中的 `appKey` 值。

```ts
import config from '@adonisjs/core/services/config'

config.get('app.appKey')
config.get('app.http.cookie') // 读取嵌套值
```

`config.get` 方法接受一个点分隔的键，并按如下方式解析它：

  * 第一部分是您想要读取值的文件名。即 `app.ts` 文件。
  * 字符串的其余部分是您想要从导出值中访问的键。例如，本例中的 `appKey`。

### 配置服务 vs. 直接导入配置文件 (Config service vs. directly importing config files)

使用配置服务而非直接导入配置文件并没有直接的好处。然而，配置服务是**唯一**可以在外部软件包和 Edge 模板中读取配置的选择。

### 在外部软件包中读取配置 (Reading config inside external packages)

如果您正在创建一个第三方软件包，您不应该直接从用户应用程序中导入配置文件，因为它会使您的软件包与宿主应用程序的文件夹结构紧密耦合。

相反，您应该在\*\*服务提供者 (service provider)\*\*内部使用配置服务来访问配置值。例如：

```ts
import { ApplicationService } from '@adonisjs/core/types'

export default class DriveServiceProvider {
  constructor(protected app: ApplicationService) {}
  
  register() {
    this.app.container.singleton('drive', () => {
      // highlight-start
      const driveConfig = this.app.config.get('drive')
      return new DriveManager(driveConfig)
      // highlight-end
    })
  }
}
```

### 在 Edge 模板中读取配置 (Reading config inside Edge templates)

您可以使用 `config` 全局方法在 Edge 模板中访问配置值。

```edge
<a href="{{ config('app.appUrl') }}"> Home </a>
```

您可以使用 `config.has` 方法来检查给定键的配置值是否存在。如果该值为 `undefined`，则此方法返回 `false`。

```edge
@if(config.has('app.appUrl'))
  <a href="{{ config('app.appUrl') }}"> Home </a>
@else
  <a href="/"> Home </a>
@end
```

## 更改配置位置 (Changing the config location)

您可以通过修改 [`adonisrc.ts`](https://www.google.com/search?q=../concepts/adonisrc_file.md) 文件来更新配置目录的位置。更改后，配置文件将从新位置导入。

```ts
directories: {
  config: './configurations'
}
```

请确保更新 `package.json` 文件中的导入别名 (import alias)。

```json
{
  "imports": {
    "#config/*": "./configurations/*.js"
  }
}
```

## 配置文件的限制 (Config files limitations)

存储在 `config` 目录中的配置文件是在应用程序的\*\*引导阶段 (boot phase)\*\*导入的。因此，配置文件不能依赖于应用程序代码。

例如，如果您尝试在 `config/app.ts` 文件中导入并使用路由服务 (router service)，应用程序将无法启动。这是因为路由服务直到应用程序处于 `booted`（已引导）状态时才会被配置。

从根本上说，这种限制对您的代码库产生了积极影响，因为应用程序代码应该依赖于配置，而不是反之。

## 运行时更新配置 (Updating config at runtime)

您可以使用配置服务在运行时更改配置值。`config.set` 会更新内存中的值，但不会对磁盘上的文件进行任何更改。

:::note

配置值是针对**整个应用程序**进行更改的，而不仅仅针对单个 HTTP 请求。这是因为 Node.js 不是一个线程化的运行时环境，并且 Node.js 中的内存是在多个 HTTP 请求之间共享的。

:::

```ts
import env from '#start/env'
import config from '@adonisjs/core/services/config'

const HOST = env.get('HOST')
const PORT = env.get('PORT')

config.set('app.appUrl', `http://${HOST}:${PORT}`)
```
