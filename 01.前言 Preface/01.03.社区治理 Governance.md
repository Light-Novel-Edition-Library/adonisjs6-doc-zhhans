# 治理 (Governance)

## 角色和责任 (Roles and responsibilities)

### 作者 (Authors)

Harminder Virk（AdonisJS 的创建者）担任项目作者 (Project Author)。项目作者负责项目的治理、标准和方向。总结如下：

* 项目作者决定哪些新项目应归入 AdonisJS 旗下。
* 项目作者负责为项目指派负责人，并在现任负责人卸任时将项目移交给新负责人。
* 作者有责任分享/记录框架的愿景，并确保项目负责人与之保持同步。

### 项目负责人 (Project Leads)

AdonisJS 是由核心团队创建和管理的多个软件包的组合。所有这些软件包都由项目作者选定的项目负责人 (Project Lead) 领导。

在几乎所有情况下，软件包的创建者都会担任项目负责人，因为他们是最初付出努力将想法变为现实的人。

项目负责人对项目内的所有决策拥有最终决定权。但是，由于社区始终拥有分叉 (fork) 的能力，因此该负责人要完全对社区负责。项目负责人的责任是设定项目的战略目标，并将其清晰地传达给社区。他们还必须了解整个社区，并努力满足尽可能多的相互冲突的需求，同时确保项目能够长期生存。

在许多方面，项目负责人的角色是关于外交的。关键是要确保随着项目的扩展，合适的人被赋予对项目的影响力，并且社区能够团结在项目负责人的愿景周围。负责人的工作随后是确保核心团队成员（见下文）代表项目做出正确的决定。一般来说，只要核心团队成员与项目的战略保持一致，项目负责人就会允许他们按意愿行事。

项目负责人**不能**归档或决定将项目从 AdonisJS 旗下移除。他们可以决定停止在该项目上的工作，在这种情况下，我们将寻找新的项目负责人。

### 核心团队 (Core team)

核心团队成员是为项目做出多次有价值贡献的贡献者，现在被信赖可以直接向代码库编写代码并筛选他人的贡献。在许多情况下，他们是程序员，但他们也可能以不同的角色做出贡献，例如社区参与。通常，核心团队成员会专注于项目的特定方面，并带来一定水平的专业知识和理解，从而赢得社区和项目负责人的尊重。核心团队成员的角色不是官方的，它只是社区中有影响力的成员在项目负责人向其寻求指导和支持时所处的地位。

核心团队成员对项目的总体方向没有权力。但是，他们确实能让项目负责人倾听他们的意见。核心团队成员的工作是确保负责人了解社区的需求和集体目标，并帮助开发或通过启发获得对项目的适当贡献。通常，核心团队成员会被赋予对其特定职责范围的非正式控制权，并被分配直接修改源代码某些区域的权利。也就是说，虽然核心团队成员没有明确的决策权，但他们经常会发现他们的行动与负责人的决定是一致的。

#### 活跃核心团队成员 (Active Core Team Members)

活跃核心团队成员定期为项目做出贡献。活跃核心团队成员通常有一个或多个关注领域——在最常见的情况下，他们将负责子项目仓库中的定期问题分类、错误修复、文档改进或功能开发。

#### 核心团队荣誉成员 (Core Team Emeriti)

一些过去做出过有价值贡献的核心团队成员，由于各种原因，今天可能无法再投入同等水平的参与。这是完全正常的，对项目过去的任何贡献仍然受到高度赞赏。这些核心团队成员因其贡献而被授予核心团队荣誉成员的称号，并欢迎随时恢复积极参与。

### 贡献者 (Contributors)

贡献者是那些要么不想成为核心团队成员，要么尚未被项目负责人给予机会的社区成员。他们做出了有价值的贡献，例如下面列表中概述的那些，但通常没有直接更改项目代码的权力。贡献者通过沟通工具参与项目，例如 RFC 讨论、GitHub issues 和 pull requests、Discord 聊天室以及论坛。

任何人都可以成为贡献者。对项目没有承诺期望，没有特定的技能要求，也没有选拔过程。要成为贡献者，社区成员只需执行一项或多项对项目有益的行动。

一些贡献者可能已经作为用户参与项目，但也会发现自己正在做以下一项或多项工作：

* 支持新用户（现有用户通常能提供最有效的新用户支持）
* 报告 Bug
* 识别需求
* 编程
* 协助项目基础设施
* 修复 Bug
* 添加功能

随着贡献者获得经验并熟悉项目，他们可能会发现项目负责人开始越来越依赖他们。当这种情况开始发生时，他们会逐渐承担起上述核心团队成员的角色。

### 用户 (Users)

用户是有项目需求的社区成员。他们是社区中最重要的成员：没有他们，项目就没有目的。任何人都可以成为用户；没有特定要求。

应鼓励用户尽可能多地参与项目和社区的生活。用户贡献使项目团队能够确保他们正在满足这些用户的需求。常见的用户活动包括（但不限于）：

* 宣传项目。
* 从新用户的角度告知开发者项目的优势和劣势。
* 提供精神支持（一句“谢谢”大有帮助）。
* 通过 GitHub Sponsors 提供经济支持。

继续参与项目及其社区的用户通常会发现自己越来越投入。这些用户随后可能会继续成为贡献者，如上所述。

## 支持 (Support)

鼓励社区中的所有参与者在项目管理基础设施内为新用户提供支持。这种支持是发展社区的一种方式。寻求支持的人应该认识到，项目内的所有支持活动都是自愿的，因此是在时间允许的情况下提供的。如果用户需要有保证的响应时间或结果，则应寻求购买支持合同。但是，对于那些愿意按项目条款参与并愿意帮助支持其他用户的用户来说，社区支持渠道是理想的选择。

## 金钱捐赠 (Monetary Donations)

对于一个开放开发项目来说，金钱不如积极贡献重要。然而，有些个人或组织资金充裕但时间匮乏，他们更愿意以现金形式做出贡献。如果您想进行大额捐赠，您可以赞助我们实施新功能或修复一些 Bug。项目网站提供了关于如何进行捐赠的明确指南。

如果您经营着一家将该项目作为创收产品的企业，那么赞助其开发在商业上是有意义的。这可以确保您的产品所依赖的项目保持健康并得到积极维护。它还可以提高我们在社区中的曝光率，并使吸引新开发人员变得更容易。

## 品牌和所有权 (Branding and Ownership)

AdonisJS（末尾拼写为 "JS"）是 Harminder Virk 的注册商标。

只有 **@adonisjs** npm 作用域和 **AdonisJS** GitHub 组织下的项目才由核心团队管理和官方支持。

此外，您不得以可能错误暗示与 AdonisJS 有任何官方联系或认可的方式使用 AdonisJS 名称或徽标。不允许以任何可能导致客户混淆的方式使用 AdonisJS 名称或徽标。

这包括以强调 AdonisJS 品牌的方式命名产品或服务，例如 "AdonisJS UIKit" 或 "AdonisJS Studio"，以及在域名中使用，如 "adonisjs-studio.com"。

相反，您必须使用您自己的品牌名称，并将其与 AdonisJS 明确区分开来。

此外，未经明确书面同意，您不得将我们的商标用于 T 恤、贴纸或其他商品。

### AdonisJS 旗下的项目 (Projects under AdonisJS umbrella)

AdonisJS 旗下的项目是项目作者的知识产权。一旦项目负责人创建的项目成为 "AdonisJS GitHub 组织" 的一部分，或者如果它是在 @adonisjs npm 作用域下发布的，项目负责人就不能删除或放弃该项目。
