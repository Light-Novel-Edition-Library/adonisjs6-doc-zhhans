# 文件夹结构 (Folder structure)

在本指南中，我们将带您浏览 AdonisJS 在安装过程中创建的重要文件和文件夹。

我们提供了一个经过深思熟虑的默认文件夹结构，可帮助您保持项目整洁且易于重构。但是，您拥有完全的自由去改变结构，以适应您的团队和项目的需求。

## `adonisrc.ts` 文件

`adonisrc.ts` 文件用于配置**工作区**和应用程序的一些**运行时设置**。

在此文件中，您可以注册服务提供者 (providers)、定义命令别名，或指定要复制到生产构建 (production build) 的文件。

另请参阅：[AdonisRC 文件参考指南](https://www.google.com/search?q=../concepts/adonisrc_file.md)

-----

## `tsconfig.json` 文件

`tsconfig.json` 文件存储应用程序的 **TypeScript 配置**。您可以根据项目或团队的要求随意更改此文件。

以下配置选项是 AdonisJS 内部机制正常工作所必需的：

```json
{
  "compilerOptions": {
    "module": "NodeNext",
    "isolatedModules": true,
    "declaration": false,
    "outDir": "./build",
    "esModuleInterop": true,
    "experimentalDecorators": true,
    "emitDecoratorMetadata": true,
    "skipLibCheck": true
  }
}
```

-----

## 子路径导入 (The sub-path imports)

AdonisJS 使用 Node.js 的[子路径导入 (sub-path imports)](https://nodejs.org/dist/latest-v19.x/docs/api/packages.html#subpath-imports) 功能来定义**导入别名**。

以下导入别名已在 `package.json` 文件中预配置。您可以随意添加新的别名或编辑现有的别名。

```json
// title: package.json
{
  "imports": {
    "#controllers/*": "./app/controllers/*.js",
    "#exceptions/*": "./app/exceptions/*.js",
    "#models/*": "./app/models/*.js",
    "#mails/*": "./app/mails/*.js",
    "#services/*": "./app/services/*.js",
    "#listeners/*": "./app/listeners/*.js",
    "#events/*": "./app/events/*.js",
    "#middleware/*": "./app/middleware/*.js",
    "#validators/*": "./app/validators/*.js",
    "#providers/*": "./app/providers/*.js",
    "#policies/*": "./app/policies/*.js",
    "#abilities/*": "./app/abilities/*.js",
    "#database/*": "./database/*.js",
    "#tests/*": "./tests/*.js",
    "#start/*": "./start/*.js",
    "#config/*": "./config/*.js"
  }
}
```

## `bin` 目录

`bin` 目录包含在特定环境中加载应用程序的**入口文件**。例如：

  * `bin/server.ts` 文件在 Web 环境中引导应用程序以监听 HTTP 请求。
  * `bin/console.ts` 文件引导 Ace 命令行并执行命令。
  * `bin/test.ts` 文件引导应用程序以运行测试。

-----

## `ace.js` 文件

`ace` 文件引导您应用程序本地的命令行框架。因此，您每次运行一个 Ace 命令时，它都会经过此文件。

如果您注意到，`ace` 文件以 `.js` 扩展名结尾。这是因为我们希望在不编译它的情况下使用 `node` 二进制文件运行此文件。

-----

## `app` 目录

`app` 目录用于组织应用程序的**领域逻辑**代码。例如，控制器 (controllers)、模型 (models)、服务 (services) 等都位于 `app` 目录中。

您可以随意创建额外的目录，以更好地组织您的应用程序代码。

```
├── app
│  └── controllers
│  └── exceptions
│  └── middleware
│  └── models
│  └── validators
```

-----

## `resources` 目录

`resources` 目录包含 Edge 模板，以及前端代码的源文件。换句话说，应用程序**表示层**的代码位于 `resources` 目录中。

```
├── resources
│  └── views
│  └── js
│  └── css
│  └── fonts
│  └── images
```

-----

## `start` 目录

`start` 目录包含您希望在应用程序**引导生命周期**中导入的文件。例如，用于注册路由和定义**事件触发器** (event listeners) 的文件应该位于 `start` 目录中。

```
├── start
│  ├── env.ts
│  ├── kernel.ts
│  ├── routes.ts
│  ├── validator.ts
│  ├── events.ts
```

AdonisJS 不会自动从 `start` 目录导入文件。它仅仅被用作对类似文件进行分组的**约定**。

我们建议阅读有关[预加载文件](https://www.google.com/search?q=../concepts/adonisrc_file.md%23preloads)和[应用程序引导生命周期](https://www.google.com/search?q=../concepts/application_lifecycle.md)的内容，以便更好地理解哪些文件应该放在 `start` 目录下。

-----

## `public` 目录

`public` 目录托管**静态资产**，例如 CSS 文件、图像、字体或前端 JavaScript。

不要将 `public` 目录与 `resources` 目录混淆。`resources` 目录包含前端应用程序的**源代码**，而 `public` 目录包含**编译后的输出**。

当使用 Vite 时，您应该将前端资产存储在 `resources/<SUB_DIR>` 目录中，并让 Vite 编译器在 `public` 目录中创建输出。

另一方面，如果您不使用 Vite，您可以直接在 `public` 目录中创建文件，并使用文件名访问它们。例如，您可以通过 `http://localhost:3333/style.css` URL 访问 `./public/style.css` 文件。

-----

## `database` 目录

`database` 目录包含用于**数据库迁移 (migrations)** 和**数据填充 (seeders)** 的文件。

```
├── database
│  └── migrations
│  └── seeders
```

-----

## `commands` 目录

[Ace 命令](https://www.google.com/search?q=../ace/introduction.md)存储在 `commands` 目录中。您可以通过运行 `node ace make:command` 在此文件夹中创建命令。

-----

## `config` 目录

`config` 目录包含应用程序的**运行时配置文件**。

框架核心和其他已安装的软件包从此目录读取配置文件。您也可以将应用程序本地的配置存储在此目录中。

了解更多关于[配置管理](https://www.google.com/search?q=./configuration.md)的信息。

```
├── config
│  ├── app.ts
│  ├── bodyparser.ts
│  ├── cors.ts
│  ├── database.ts
│  ├── drive.ts
│  ├── hash.ts
│  ├── logger.ts
│  ├── session.ts
│  ├── static.ts
```

-----

## `types` 目录

`types` 目录是应用程序中使用的 **TypeScript 接口或类型**的所在地。

该目录默认是空的，但是您可以在 `types` 目录中创建文件和文件夹来定义自定义类型和接口。

```
├── types
│  ├── events.ts
│  ├── container.ts
```

-----

## `providers` 目录

`providers` 目录用于存储应用程序使用的[服务提供者](https://www.google.com/search?q=../concepts/service_providers.md)。您可以使用 `node ace make:provider` 命令创建新的提供者。

了解更多关于[服务提供者](https://www.google.com/search?q=../concepts/service_providers.md)的信息。

```
├── providers
│  └── app_provider.ts
│  └── http_server_provider.ts
```

-----

## `tmp` 目录

应用程序生成的**临时文件**存储在 `tmp` 目录中。例如，这些文件可能是用户上传的文件（在开发过程中生成）或写入磁盘的日志。

`tmp` 目录必须被 `.gitignore` 规则忽略，并且您也不应该将其复制到生产服务器。

-----

## `tests` 目录

`tests` 目录用于组织您的应用程序测试。它进一步创建了用于**单元测试 (unit)** 和**功能测试 (functional)** 的子目录。

另请参阅：[测试](https://www.google.com/search?q=../testing/introduction.md)

```
├── tests
│  ├── bootstrap.ts
│  └── functional
│  └── regression
│  └── unit
```
